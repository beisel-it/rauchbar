Deal lifecycle

Shared lifecycle grammar across site, member, and admin.

discovered -> normalized -> in_review -> approved -> member_visible -> public_visible -> expired

Public site

Homepage and archive states

  • public_visible full card with public timestamp
  • expired preserved shell with unavailable copy
  • empty explain member-first delay and keep signup CTA
  • error retry-friendly copy without losing primary actions

Member surface

Member home state stack

  • sent latest digest delivered
  • pending_verification WhatsApp channel needs action
  • partial preferences can be tuned for better relevance
  • scheduled next digest queued

Admin operations

Operational severity and queue treatment

  • warning crawler delayed, intervention may be needed soon
  • critical send failure requires operator action
  • held_for_review item needs manual quality check
  • healthy shop and pipeline are within normal bounds

Badge guidance

Semantic groupings

success warning danger neutral

Keep wording identical across surfaces even if the visual treatment differs.
